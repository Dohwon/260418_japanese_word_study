# JLPT Japanese Word Study

JLPT 단어를 카드 덱 방식으로 학습하고, 틀린 단어와 아카이브 단어를 다시 복습할 수 있는 웹 앱입니다.  
현재 프로젝트는 Railway에 배포되어 있으며, Google 로그인으로 웹과 모바일 사이의 학습 상태를 동기화합니다.

## 구성

- 메인 화면
  - `단어 공부하기`
  - `전체 오답 확인하기`
- 단어 공부하기
  - `공부하기`
  - `아카이브`
- 전체 단어 확인하기
  - `진행중`
  - `완료`

## 현재 데이터 범위

- 현재 앱 데이터는 `N5`, `N4`, `N3`가 반영되어 있습니다.
- 원본 자료는 `DATA/` 폴더의 JLPT 급수별 PDF 및 추출 데이터 기준으로 정리했습니다.
- 기본 노출 순서는 프로젝트 데이터의 원래 순서를 최대한 유지합니다.

## 단어 선정 기준

이 앱은 한 번에 무작위로만 섞는 방식이 아니라, "처음 보는 카드", "자주 틀린 카드", "이미 어느 정도 익힌 카드"를 섞어 한 묶음씩 학습하도록 설계했습니다.

### 기본 학습 묶음 규칙

`공부 시작하기`를 누르면 1회 학습 묶음은 최대 30장으로 구성됩니다.

- 처음 배우는 카드 10장
- 틀렸던 카드 7장
- 1회 맞춘 카드 5장
- 2회 맞춘 카드 3장
- 3회 맞춘 카드 3장
- 4회 이상 맞춘 카드 2장

### 카드 분류 기준

- 처음 배우는 카드
  - `studyCount === 0`
- 틀렸던 카드
  - 틀린 기록이 있고, 실질적으로 아직 완전히 익히지 못한 카드
- 맞춘 카드 버킷
  - `유효 맞춤 횟수 = correctHits - wrongHits`
  - 예를 들어 3번 맞췄어도 2번 틀렸다면 `1회 맞춘 카드`처럼 취급합니다.

이 기준을 둔 이유는 단순 정답 횟수만 보면 실제 체감 숙련도보다 높게 잡히기 때문입니다.  
틀린 이력이 많을수록 더 낮은 버킷으로 다시 내려와 재노출되게 했습니다.

### 노출 우선순위

- 오답 카드는 `wrongHits`가 높은 카드부터 우선합니다.
- 랜덤 모드에서도 학습량이 특정 카드에만 쏠리지 않도록, 대체로 덜 나왔던 카드와 오래 안 나온 카드가 먼저 오게 구성했습니다.
- 기본 모드에서는 현재 진행 중인 급수와 챕터 순서를 최대한 유지합니다.

## 의미 판정 기준

뜻 맞추기는 완전 일치만 보지 않습니다.

- 부분 포함
- 조사 제거
- 사전형 동의어 치환
- 서버 임베딩 기반 의미 유사도 판정

예를 들어 아래 같은 표현은 정답 처리될 수 있도록 설계했습니다.

- `꽃병` ↔ `화병`
- `소고기` ↔ `쇠고기`
- `나라` ↔ `국가`

즉, 사용자가 실제로 뜻을 알고 있는데 표현만 조금 다른 경우를 최대한 오답으로 치지 않도록 조정했습니다.

## 아카이브 규칙

- 4회 이상 익힌 단어는 완료/아카이브 영역으로 이동합니다.
- 아카이브 모드에서는 무작위 30문제를 시간 기록과 함께 풉니다.
- 같은 카드를 3번 이상 틀리면 다시 일반 학습 목록으로 내려갑니다.
- 카드가 하나도 없으면 아카이브 타이머는 시작되지 않습니다.
- 진행 중에는 일시정지/재개가 가능합니다.

## 반응형 처리

이 프로젝트는 모바일에서 "카드 한 장 + 좌우 더미" 구조가 무너지지 않도록 배치를 단계적으로 바꿉니다.

### 데스크톱

- `오답 더미 | 현재 카드 | 정답 더미`
- 비율은 중앙 카드가 더 크고, 좌우 더미가 작은 구조입니다.

### 태블릿 이하

- 카드가 위로 올라가고
- 아래 줄에 `오답 더미 | 정답 더미` 2열로 내려옵니다.

### 모바일

- 화면 폭이 더 좁아지면
  - `현재 카드`
  - `오답 더미`
  - `정답 더미`
- 순서로 세로 배치됩니다.

즉, 모바일에서 좌우 더미가 사라지는 대신, 아래쪽으로 재배치해서 항상 보이도록 처리했습니다.  
더미 카드 자체의 높이와 글자 크기도 함께 줄여 작은 화면에서 잘리지 않게 맞췄습니다.

## 로그인 / 동기화

- Google 로그인 버튼은 웹과 모바일 모두 같은 방식으로 동작합니다.

- Railway 볼륨에 데이터를 저장해 재배포 후에도 상태가 유지되도록 구성했습니다.
